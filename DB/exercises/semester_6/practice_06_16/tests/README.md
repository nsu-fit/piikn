# Контракт проверки

Автоматически можно проверить:

- наличие обязательных файлов;
- наличие ссылок на checkpoint-работы;
- наличие demo-сценария или design walkthrough;
- наличие разделов risks и alternatives.

Вручную проверяются:

- инженерная конкретность проблемы;
- воспроизводимость или обоснованность design slice;
- ответы на защите;
- связь с теоретическими темами экзамена.
