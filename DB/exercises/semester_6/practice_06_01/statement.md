# Практика 6.1. Локальное или контейнерное развертывание PostgreSQL

Вы поднимаете собственный учебный PostgreSQL-контур для всего 6-го семестра. Дальнейшие практики должны воспроизводиться в этом окружении.

## Что нужно сдать

```text
semester-6/practice-01-postgres-local-setup/
  README.md
  compose.yml или setup.md
  init.sql
  seed.sql
  checks.sql
```

## Требования к окружению

- PostgreSQL версии не ниже `16`.
- Отдельная база `db_course`.
- Роли:
  - `db_course_owner` для миграций и владения объектами;
  - `db_course_app` для прикладных запросов;
  - `db_course_readonly` для чтения.
- Учебная схема `app`.
- Seed-таблицы из `provided/seed_operational_data.sql`.

## Задания

1. Подготовьте способ запуска: `compose.yml` для Docker/Podman или `setup.md` для локальной установки.
2. В `init.sql` создайте базу, роли, схему и права.
3. В `seed.sql` подключите или скопируйте выданный seed dataset.
4. В `checks.sql` добавьте smoke checks:
   - версия PostgreSQL;
   - текущая база и пользователь;
   - список ролей курса;
   - таблицы схемы `app`;
   - количество строк в `app.events`, `app.orders`, `app.documents`.
5. В `README.md` опишите запуск, остановку, очистку и команды, которые удаляют данные.
