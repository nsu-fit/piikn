# Контракт проверки

Автоматически проверяются:

- наличие обязательных файлов;
- наличие `PARTITION BY` или явного design-only обоснования;
- наличие retention policy;
- наличие запроса с фильтром по `created_at`;
- наличие минимум двух альтернатив в ADR.

Вручную проверяются:

- уместность partition key;
- реалистичность retention;
- понимание trade-off.
