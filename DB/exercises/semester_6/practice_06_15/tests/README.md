# Контракт проверки

Автоматически проверяются:

- наличие обязательных файлов;
- наличие `to_tsvector` и `tsquery` в SQL-прототипе или design-only обоснования;
- наличие минимум 3 тестовых запросов;
- наличие tenant/access boundary в описании.

Вручную проверяются:

- качество search design;
- реалистичность pipeline обновления;
- оценка рисков AI/vector-сценариев.
