# Контракт проверки

Автоматически проверяются:

- наличие обязательных файлов;
- наличие одного из паттернов: `ON CONFLICT`, `FOR UPDATE SKIP LOCKED` или keyset-фильтра без `OFFSET`;
- отсутствие небезопасной строковой сборки пользовательских значений;
- наличие минимум 3 тестовых сценариев в `test-sketch.md`.

Вручную проверяются:

- корректность транзакционной границы;
- связь паттерна с прикладной проблемой;
- качество обработки ошибок.
