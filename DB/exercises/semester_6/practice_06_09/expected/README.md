# Ожидаемые свойства решения

В корректной работе:

- выбранный паттерн связан с конкретным use case;
- транзакционная граница описана явно;
- SQL использует параметры для пользовательских значений;
- тесты проверяют поведение, а не только наличие функции;
- N+1 или избыточные запросы обсуждены применительно к сценарию.
