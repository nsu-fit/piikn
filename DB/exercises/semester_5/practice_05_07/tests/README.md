# Контракт автопроверки

Автоматически проверяются:

- наличие `README.md`, `changes.sql`, `checks.sql`, `answers.md`;
- выполнение `provided/seed_dml_lab.sql`;
- выполнение `changes.sql` без ошибок;
- повторное выполнение `changes.sql` без нарушения уникальности audit-событий;
- выполнение `checks.sql` без ошибок;
- отсутствие физического `DELETE FROM course_lab.order_queue`;
- наличие `RETURNING` в DML-сценарии;
- наличие `UPDATE ... FROM` или эквивалентного корректного DML на основе `SELECT`.

Вручную проверяется:

- точность предварительного `SELECT`;
- объяснение идемпотентности;
- интерпретация проверочных запросов;
- понимание разницы между мягкой отменой и физическим удалением.
