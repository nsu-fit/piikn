# Контракт проверки

Автоматически проверяются:

- наличие `README.md`, `answers.md`, `solution.sql`, `decomposition.sql`;
- наличие меток `task_a`, `task_b`, `task_c`, `task_d`;
- отсутствие обращений к внешним файлам, не входящим в вариант;
- выполнение SQL-фрагментов, если вариант содержит исполняемую учебную схему.

Вручную проверяются:

- корректность зависимостей и ключей;
- качество декомпозиции;
- разбор транзакционного timeline;
- безопасность исправленного SQL;
- ответы на короткой устной проверке.
