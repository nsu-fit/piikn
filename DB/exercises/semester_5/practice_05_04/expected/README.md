# Ожидаемые свойства решения

В корректной работе:

- назначение таблиц объяснено через предметную область;
- первичные и внешние ключи не перепутаны с обычными атрибутами;
- кардинальности связей обоснованы схемой или явно помечены как гипотезы;
- готовые диагностические выводы интерпретированы как проверки конкретных правил;
- студент отличает "на текущих данных нарушений не найдено" от "схема гарантирует невозможность нарушения";
- ER-фрагмент согласован с текстовым ответом.

Не требуется:

- писать собственные SQL-запросы;
- восстанавливать всю схему демобазы;
- знать внутреннее устройство PostgreSQL.
