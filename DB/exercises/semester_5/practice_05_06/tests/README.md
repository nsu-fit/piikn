# Контракт автопроверки

Автоматически проверяются:

- наличие `README.md`, `queries.sql`, `answers.md`;
- наличие меток `q01_...` - `q07_...`;
- выполнение `queries.sql` без ошибок на базе `3 months`;
- наличие явного `ORDER BY` в запросах с `LIMIT`;
- отсутствие hard-code конкретных `ticket_no`, `book_ref`, `flight_id`, если они не входят в вариант;
- переносимость на hidden-базу `6 months` для части запросов.

Для каждого запроса можно проверять:

- количество и имена столбцов;
- что результат не пустой там, где вариант должен давать строки;
- что `q04`, `q05`, `q07` действительно агрегируют данные;
- что `q06` не использует ручной список аэропортов.

Вручную проверяются:

- объяснение неправильного join в `q08`;
- читаемость SQL;
- корректность выбранной цепочки соединений.
