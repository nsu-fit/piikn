# Контракт проверки

## Автоматические проверки

- Проверить наличие `README.md`, `er.md`, `q01_keys.sql`, `q02_relationship_checks.sql`, `q03_missing_constraints.sql`, `answers.md`.
- Выполнить все SQL-файлы.
- Проверить, что `q02_relationship_checks.sql` содержит минимум 4 комментария `-- check:`.
- Проверить, что `q03_missing_constraints.sql` содержит минимум 2 проверки.
- Проверить, что запросы с `LIMIT` имеют `ORDER BY`.

## Hidden checks

- Запустить SQL на базе `6 months`.
- Проверить, что relationship checks не завязаны на конкретные `flight_id`, `ticket_no`, `book_ref`, если они не выданы как параметр варианта.

## Ручная проверка

- ER-фрагмент проверяется вручную.
- Ключевой критерий: студент должен объяснить, чем подтверждена каждая связь.
- На защите преподаватель выбирает одну связь и просит показать SQL-доказательство.
