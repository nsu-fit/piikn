# Ожидаемые свойства решения

## relationship_checks.sql

- Совпадает с выданным `provided/relationship_checks.sql` или отличается только комментариями студента.
- Выполняется без ошибок.

## er.md

- Есть ER/EER-фрагмент или текстовая эквивалентная модель.
- Для каждой связи указаны кардинальность и основание: constraint, diagnostic query, documentation или domain assumption.
- Спорные места явно отмечены.

## relationship_evidence.md

- Описаны минимум 4 диагностические проверки.
- Для каждой проверки указано, что она подтверждает или не подтверждает.
- Отдельно выделены минимум 2 правила, которые не полностью гарантируются схемой.

## answers.md

- Ответы различают "ограничение схемы", "наблюдение на данных" и "гипотезу предметной области".
