# Ожидаемые свойства решения

## er.md

- Есть ER/EER-фрагмент или текстовая эквивалентная модель.
- Для каждой связи указаны кардинальность и основание: constraint, SQL observation или domain assumption.
- Спорные места явно отмечены.

## q01_keys.sql

- Показывает primary key, foreign key и unique constraints для таблиц варианта.
- Использует системные каталоги или information schema.

## q02_relationship_checks.sql

- Содержит минимум 4 проверки.
- Каждая проверка имеет комментарий `-- check: <name>`.
- Проверки помогают подтвердить кардинальности, а не просто считают строки таблиц.

## q03_missing_constraints.sql

- Содержит минимум 2 проверки правил, которые не полностью гарантируются схемой.
- В комментарии к каждой проверке указано проверяемое правило.

## answers.md

- Ответы различают "ограничение схемы", "наблюдение на данных" и "гипотезу предметной области".
