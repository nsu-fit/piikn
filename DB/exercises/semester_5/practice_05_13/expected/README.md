# Ожидаемые свойства решения

В корректной работе:

- сценарии разделены на шаги для двух сессий;
- транзакции явно начинаются и завершаются;
- в журнале есть фактические наблюдения, а не только теория;
- lost update объяснен как следствие записи устаревшего прикладного значения;
- конфликт или deadlock не маскируется, а описывается с безопасной реакцией.

Не требуется разбирать структуру tuple versions, clog или внутренние алгоритмы блокировок PostgreSQL.
