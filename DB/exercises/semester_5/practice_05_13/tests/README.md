# Контракт проверки

Автоматически можно проверить:

- наличие `README.md`, `setup.sql`, `session_a.sql`, `session_b.sql`, `experiment_log.md`;
- выполнение `setup.sql` без ошибок;
- наличие в сессионных файлах `BEGIN` и `COMMIT` или `ROLLBACK`;
- наличие меток шагов `A1`/`B1`;
- отсутствие неявного изменения таблиц вне транзакционных сценариев.

Вручную проверяются:

- корректность порядка шагов в двух сессиях;
- фактические наблюдения;
- объяснение аномалии или конфликта;
- отделение наблюдаемого поведения от внутренних деталей реализации.
