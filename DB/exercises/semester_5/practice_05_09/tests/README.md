# Контракт автопроверки

Автоматически проверяются:

- наличие `README.md`, `schema.sql`, `check_observations.md`, `answers.md`;
- выполнение `schema.sql` в чистом снапшоте;
- наличие таблиц `course_lab.issue_projects`, `course_lab.issue_tickets`, `course_lab.issue_comments`;
- наличие первичных и внешних ключей;
- наличие `CHECK` для `status`, `priority`, `closed_at`;
- выполнение `provided/check_constraints.sql` без аварийного завершения;
- что invalid-вставки действительно отклоняются.

Вручную проверяются:

- качество выбора типов данных;
- осмысленность имен ограничений;
- объяснение, какие DDL-темы сознательно оставлены на 6-й семестр;
- связь DDL-ограничений с будущей нормализацией.
