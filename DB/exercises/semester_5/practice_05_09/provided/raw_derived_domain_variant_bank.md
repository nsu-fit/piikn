# Банк предметных вариантов для DDL

Источник идей: `DB/raw/Рабочая группа РГБД 2025/Практические задания` и `Контрольная работа №2`.

Этот файл можно использовать для замены основного issue-tracker сценария в практике 5.9 или как банк вариантов для контрольной 5.15. Каждый вариант нужно сопровождать отдельным проверочным SQL, если он выдается как обязательная работа.

## Вариант DDL-LIB. Библиотека

Минимальная схема:

- `branches` - филиалы библиотеки;
- `readers` - читатели;
- `books` - библиографические описания книг;
- `book_copies` - физические экземпляры книг в филиалах;
- `loans` - выдачи экземпляров читателям.

Обязательные инварианты:

- экземпляр книги принадлежит существующей книге и филиалу;
- активная выдача экземпляра может быть только одна;
- дата возврата не раньше даты выдачи;
- год издания книги находится в разумном диапазоне;
- читатель имеет обязательные ФИО и год рождения.

Минимальные проверки:

- нельзя создать экземпляр для несуществующей книги;
- нельзя создать выдачу для несуществующего экземпляра;
- нельзя создать выдачу с `returned_at < loaned_at`;
- нельзя создать две активные выдачи одного экземпляра.

## Вариант DDL-PHARM. Аптека

Минимальная схема:

- `suppliers` - поставщики;
- `products` - лекарственные препараты;
- `pharmacy_branches` - филиалы аптечной сети;
- `stock_items` - остатки товара по филиалам;
- `sales` - продажи.

Обязательные инварианты:

- ИНН поставщика уникален;
- цена и остаток неотрицательны;
- товар связан с поставщиком;
- продажа связана с филиалом и товаром;
- количество в продаже положительное;
- признак `prescription_required` задает, нужна ли продажа по рецепту.

Минимальные проверки:

- отрицательная цена отклоняется;
- дубль ИНН поставщика отклоняется;
- продажа с количеством `0` отклоняется;
- товар без поставщика отклоняется.

## Вариант DDL-COOK. Кулинарная книга

Минимальная схема:

- `recipes` - рецепты;
- `products` - продукты;
- `recipe_components` - продукты и количество в рецепте.

Обязательные инварианты:

- название рецепта уникально;
- рецепт рассчитан минимум на одну порцию;
- продукт имеет цену, единицу измерения и калорийность;
- цена и калорийность неотрицательны;
- компонент рецепта задается парой `(recipe_id, product_id)`;
- количество продукта в рецепте положительное.

Минимальные проверки:

- нельзя добавить компонент для несуществующего рецепта;
- нельзя добавить один продукт в рецепт дважды;
- нельзя указать отрицательную цену продукта;
- нельзя создать рецепт на `0` порций.

## Вариант DDL-DELIVERY. Курьерская доставка

Минимальная схема:

- `couriers` - сотрудники-курьеры;
- `vehicles` - транспорт;
- `orders` - заказы;
- `deliveries` - назначение курьера и транспорта на заказ.

Обязательные инварианты:

- заказ имеет статус из ограниченного набора;
- доставка связана с существующим заказом, курьером и транспортом;
- один заказ не может иметь две активные доставки;
- время завершения доставки не раньше времени начала;
- транспорт имеет тип и уникальный регистрационный номер.

Минимальные проверки:

- недопустимый статус заказа отклоняется;
- доставка для несуществующего заказа отклоняется;
- дубль регистрационного номера транспорта отклоняется;
- завершение раньше начала отклоняется.
