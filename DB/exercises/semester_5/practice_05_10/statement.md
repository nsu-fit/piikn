# Практика 5.10. Функциональные зависимости и аномалии

Вы анализируете денормализованную таблицу продаж авиабилетов. SQL можно использовать для проверки гипотез на данных, но функциональные зависимости нужно формулировать из смысла предметной области.

## Что нужно сдать

```text
semester-5/practice-10-functional-dependencies/
  README.md
  answers.md
  checks.sql
  decomposition_draft.md
```

## Подготовка

1. Выполните `provided/seed_denormalized_sales.sql`.
2. Работайте с таблицей `course_lab.flight_sales_flat`.
3. В `checks.sql` пишите только диагностические `SELECT`, без изменения данных.

## Задания

### 1. Атрибуты и смысл таблицы

В `answers.md` перечислите атрибуты таблицы и кратко объясните, какую сущность или связь представляет каждый атрибут.

### 2. Функциональные зависимости

Сформулируйте минимум 6 зависимостей. Обязательные кандидаты для рассмотрения:

- `booking_ref -> book_date, total_amount`;
- `ticket_no -> booking_ref, passenger_id, passenger_name`;
- `flight_id -> route_no, departure_airport, arrival_airport, scheduled_departure`;
- `(ticket_no, flight_id) -> fare_conditions, price`.

Для каждой зависимости укажите: это правило предметной области или только совпадение в текущих данных.

### 3. Ключи

Найдите кандидатный ключ для `flight_sales_flat`. Покажите короткое рассуждение через замыкание атрибутов.

### 4. Аномалии

Опишите по одному примеру:

- аномалии обновления;
- аномалии вставки;
- аномалии удаления.

Пример должен ссылаться на конкретные атрибуты таблицы.

### 5. SQL-проверки

В `checks.sql` напишите диагностические запросы:

- `q01_flight_conflicts`: один `flight_id` связан с разными маршрутами, аэропортами или временем;
- `q02_ticket_conflicts`: один `ticket_no` связан с разными бронированиями или пассажирами;
- `q03_duplicate_segments`: повтор `(ticket_no, flight_id)`;
- `q04_booking_total_delta`: расхождение `total_amount` и суммы `price` по бронированию.

Каждый запрос должен возвращать только подозрительные строки. Если нарушений нет, результат должен быть пустым.

### 6. Черновик декомпозиции

В `decomposition_draft.md` предложите набор таблиц для следующей практики. Для каждой таблицы укажите ключ и 2-4 основных атрибута.

## Контрольные вопросы

1. Почему функциональная зависимость не доказывается только текущими строками?
2. Почему `(ticket_no, flight_id)` является естественным кандидатом на ключ сегмента?
3. Какая аномалия возникает, если аэропорт рейса хранится в каждой строке продажи?
4. Что именно проверяет каждый диагностический SQL-запрос?
