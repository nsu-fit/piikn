# Ожидаемые свойства решения

В корректной работе:

- оба представления создаются в `course_lab` и читают данные из `bookings`;
- имена столбцов соответствуют контракту задания;
- функция возвращает числовую дельту для существующего бронирования и `NULL` для отсутствующего;
- `usage.sql` демонстрирует повторное использование view и функции;
- README объясняет границы логики БД.

Не требуется создавать materialized view.
