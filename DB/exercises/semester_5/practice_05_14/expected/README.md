# Ожидаемые свойства решения

В корректной работе:

- пользовательские значения передаются параметрами, а не конкатенацией;
- динамические имена столбцов и направление сортировки выбираются из whitelist;
- tenant boundary фиксируется в SQL-предикате и объясняется как вопрос авторизации;
- роли разделяют миграции, обычную работу приложения и read-only поддержку;
- payload из варианта не меняет структуру итогового SQL.

Не требуется писать полноценное приложение или подключаться к драйверу конкретного языка.
