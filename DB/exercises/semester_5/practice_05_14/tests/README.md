# Контракт проверки

Автоматически можно проверить:

- наличие `README.md`, `secure_sql.md`, `fixed_queries.sql`;
- наличие слов `placeholder`, `whitelist`, `tenant_id` или их русских эквивалентов в объяснении;
- отсутствие прямой вставки payload-строк в исполняемый SQL;
- наличие параметров `$1`, `$2` или явно описанных placeholders.

Вручную проверяются:

- корректность границы между параметрами и идентификаторами;
- минимальность прав ролей;
- различение SQL injection, authorization bug и ошибки бизнес-логики;
- качество объяснения для code review.
