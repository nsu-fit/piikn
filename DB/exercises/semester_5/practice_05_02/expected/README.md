# Ожидаемые свойства решения

## data_dictionary.md

- Описаны все таблицы варианта.
- Для каждой таблицы указан предметный смысл, ключи и важные ограничения.
- Есть раздел "Неявные или спорные правила".

## q01_columns.sql

- Возвращает столбцы только для таблиц варианта.
- Обязательные столбцы результата: `table_name`, `column_name`, `data_type`, `is_nullable`, `ordinal_position`.

## q02_constraints.sql

- Возвращает ограничения только для таблиц варианта.
- Обязательные столбцы результата: `table_name`, `constraint_name`, `constraint_type`, `constraint_definition`.

## q03_cardinality.sql

- Содержит минимум 3 запроса.
- Запросы показывают распределения или связи между таблицами, а не просто `count(*)` каждой таблицы.

## q04_quality_checks.sql

- Содержит минимум 3 проверки.
- Одна проверка обязательно сравнивает `bookings.total_amount` с суммой `segments.price` через `tickets`.
- Каждая проверка имеет комментарий с формулировкой проверяемого правила.
