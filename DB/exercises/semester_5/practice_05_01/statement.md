# Практика 5.1. Вход в Taidon/sqlrs и запуск готового SQL-сценария

## Контекст

Вы работаете с демонстрационной базой Postgres Pro "Авиаперевозки", версия `2025-09-01`, схема `bookings`. В обычной практике используется вариант базы `3 months`.

На этой неделе от вас **не требуется писать SQL самостоятельно**. SQL-блок курса начнется позже. Сейчас важно научиться открыть среду, запустить готовый сценарий, сохранить результат и объяснить, что вы увидели.

## Что нужно сдать

Создайте в своем репозитории папку:

```text
semester-5/practice-05-01/
  README.md
  intro_queries.sql
  run_log.md
  answers.md
```

Файл `intro_queries.sql` - это копия выданного файла `provided/intro_queries.sql` без изменений. Если вы меняли его для эксперимента, приложите измененную версию отдельно как `scratch.sql` и не используйте ее как основной результат.

## Задание

### 1. Запуск готового сценария

1. Откройте Taidon/sqlrs.
2. Подключитесь к базе `bookings`.
3. Запустите выданный файл `provided/intro_queries.sql`.
4. Сохраните краткий журнал запуска в `run_log.md`.

В журнале нужно указать:

- дату выполнения;
- среду выполнения;
- версию демобазы из результата `bookings.version()`;
- какие блоки сценария выполнились успешно;
- какие ошибки возникли, если они были.

### 2. Чтение результата

В `answers.md` ответьте:

1. Какую версию демобазы показал `bookings.version()`?
2. Какие типы объектов есть в схеме `bookings`?
3. Какие три таблицы оказались самыми большими по количеству строк?
4. Чем таблица отличается от представления на примере `airports` и `airports_data`?
5. Почему `bookings.now()` лучше использовать в демобазе вместо обычного `now()`?
6. Что делает результат выполнения SQL-сценария воспроизводимым?

### 3. README

В `README.md` опишите:

- где выполнялась работа;
- как запустить `intro_queries.sql`;
- какие файлы являются выданным SQL, а какие - вашими ответами;
- как преподаватель может повторить проверку.

## Ограничения

- Не нужно писать свои SQL-запросы.
- Скриншоты не заменяют `run_log.md` и `answers.md`.
- Не удаляйте из `intro_queries.sql` запросы, которые вам кажутся непонятными.
