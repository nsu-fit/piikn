# Ожидаемые свойства решения

В корректной контрольной работе:

- `solution.sql` содержит задачи `task_01` - `task_06`;
- `SELECT`-запросы используют корректные фильтры, соединения, агрегаты и подзапросы;
- DML-задача ограничена точным предикатом и проверяется отдельным `SELECT` или `RETURNING`;
- нет DDL, CTE, оконных функций и рекурсивных запросов;
- `answers.md` объясняет не только "что написано", но и почему результат корректен.

Оценивание соответствует `DB/practical_assessment_methodology.md`.
