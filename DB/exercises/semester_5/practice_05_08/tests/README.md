# Контракт проверки

Автоматически проверяются:

- наличие `README.md`, `solution.sql`, `answers.md`;
- наличие меток `task_01` - `task_06`;
- выполнение `solution.sql` на подготовленном варианте;
- отсутствие `CREATE TABLE`, `ALTER TABLE`, `DROP TABLE`;
- отсутствие `WITH RECURSIVE`, `OVER (` и других явно запрещенных конструкций;
- наличие предварительного `SELECT` и проверочного `SELECT` или `RETURNING` в DML-части;
- наличие `ORDER BY` при `LIMIT`.

Вручную проверяются:

- корректность join-цепочек;
- смысл агрегации;
- точность DML-предиката;
- объяснение ошибочного SQL-фрагмента;
- устный ответ по одному запросу из работы.
