# Практическое занятие 5.10. Функциональные зависимости и аномалии

Место в курсе: занятие готовит к интегрированной контрольной 3. Студент учится видеть в таблице не только текущие данные, но и ограничения, которые должны быть верны для всех допустимых состояний.

Тип занятия: практика.

## Цели

- Формулировать функциональные зависимости по описанию предметной области.
- Находить кандидатные ключи.
- Выявлять аномалии вставки, обновления и удаления.
- Подготавливать аргументы для будущей декомпозиции.

## Входные условия

- Студент знает понятия функциональной зависимости и ключа.
- Преподаватель выдает денормализованную таблицу или фрагмент схемы с намеренными проблемами.
- Разрешено использовать SQL для проверки гипотез на данных, но зависимости выводятся из смысла предметной области.

## План занятия

1. Разбор различия между зависимостью в данных и зависимостью в предметной области.
2. Формулировка зависимостей для учебной таблицы.
3. Поиск замыкания набора атрибутов.
4. Определение кандидатных ключей.
5. Поиск аномалий вставки, обновления и удаления.
6. SQL-проверки, которые помогают найти подозрительные нарушения.
7. Подготовка черновика декомпозиции для следующей практики.

## Задание

Студент должен проанализировать заданную таблицу:

- выписать функциональные зависимости;
- найти минимум один кандидатный ключ;
- описать 2-3 аномалии;
- написать SQL-проверки для подозрительных повторов или противоречий;
- предложить предварительный план декомпозиции.

## Артефакт сдачи

Папка `semester-5/practice-10-functional-dependencies`:

- `answers.md`;
- `checks.sql`;
- при необходимости короткий черновик схемы декомпозиции.

## Критерии зачета

- Зависимости обоснованы предметной областью.
- Кандидатные ключи не выбираются только по текущему набору строк.
- Аномалии описаны через конкретные сценарии изменения данных.
- SQL-проверки используются как вспомогательный инструмент, а не как единственное доказательство.

## Контрольные вопросы: что вы должны уметь после занятия

1. Что означает функциональная зависимость?
2. Почему зависимость нельзя доказывать только текущими строками таблицы?
3. Как найти кандидатный ключ через замыкание?
4. Чем аномалия обновления отличается от аномалии удаления?
5. Какие SQL-запросы помогают найти противоречивые повторяющиеся данные?
6. Как функциональные зависимости подготавливают нормализацию?
