# Практическое занятие 5.9. CTE, рекурсивные запросы и оконные функции

Место в курсе: занятие расширяет SQL-инструментарий после контрольной по базовым запросам и формирует важную часть SQL-портфолио.

Тип занятия: практика.

## Цели

- Использовать CTE для разбиения сложного запроса на понятные шаги.
- Писать рекурсивный запрос для иерархических или графоподобных данных.
- Использовать оконные функции для ранжирования, running totals и сравнения строк.
- Сравнивать читаемость и стоимость разных форм запроса.

## Входные условия

- Базовый SQL уже отработан на контрольной 2.
- В учебном датасете есть данные, пригодные для иерархий, статусов, временных рядов или рейтингов.
- Студент умеет читать результат `SELECT` и проверять его на малых примерах.

## План занятия

1. Разбор CTE как способа структурировать запрос.
2. Написание запроса с несколькими CTE.
3. Рекурсивный CTE: anchor part, recursive part, условие остановки.
4. Оконные функции: `row_number`, `rank`, `lag`, `lead`, агрегаты по окну.
5. Сравнение оконной функции с группировкой.
6. Проверка результата на небольшом подмножестве данных.
7. Выбор 1-2 запросов-кандидатов для SQL-портфолио.

## Задание

Студент должен подготовить набор расширенных SQL-запросов:

- один запрос с несколькими CTE;
- один рекурсивный запрос или эквивалентный по сложности запрос по иерархии;
- два запроса с оконными функциями;
- короткое пояснение, почему выбранная форма запроса лучше простого набора вложенных подзапросов.

Папка `semester-5/practice-09-cte-windows-recursive`:

## Критерии зачета

- Запросы корректны на учебных данных.
- CTE используются для читаемости, а не как механическая замена любого подзапроса.
- Оконные функции не смешиваются с агрегатами без понимания уровня детализации.
- Рекурсивный запрос имеет понятное условие остановки.

## Контрольные вопросы: что вы должны уметь после занятия

1. Когда CTE улучшает читаемость запроса?
2. Из каких частей состоит рекурсивный CTE?
3. Чем оконная функция отличается от `GROUP BY`?
4. Как посчитать рейтинг внутри группы?
5. Как найти предыдущее или следующее событие для строки?
6. Какие риски есть у рекурсивного запроса?
