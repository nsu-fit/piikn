# Практическое занятие 5.14. MVCC, конфликты и безопасный SQL

Место в курсе: занятие связывает наблюдаемое поведение PostgreSQL с безопасным использованием SQL из приложения и готовит интегрированную контрольную 3.

Тип занятия: практика.

## Цели

- Наблюдать прикладные эффекты MVCC: видимость строк, snapshots, конфликтующие обновления.
- Разобрать, почему блокировки и версии строк влияют на приложение.
- Исправлять небезопасную строковую сборку SQL.
- Разделять параметризуемые значения и динамические идентификаторы.

## Входные условия

- Выполнена практика по транзакциям.
- Студент знаком с базовым понятием SQL injection.
- Преподаватель выдает фрагменты псевдокода или SQL-шаблонов.

## План занятия

1. Краткое повторение MVCC на уровне видимости строк.
2. Эксперимент с видимостью изменений до и после `COMMIT`.
3. Эксперимент с конфликтующим обновлением.
4. Разбор прикладного симптома: зависшая операция, неожиданный конфликт, устаревшее чтение.
5. Разбор небезопасного SQL-фрагмента.
6. Параметризация значений.
7. Проверка динамического SQL: whitelist для имен таблиц, столбцов и направлений сортировки.
8. Подготовка к контрольной 3.

## Задание

Студент должен выполнить один MVCC-эксперимент и исправить 2-3 небезопасных SQL-шаблона.

Обязательные элементы:

- сценарий двух сессий;
- описание наблюдаемой видимости изменений;
- безопасный вариант запроса с параметрами;
- объяснение, что нельзя параметризовать обычным placeholder-ом;
- краткое описание минимальных прав приложения.

## Артефакт сдачи

Папка `semester-5/practice-14-mvcc-sql-security`:

- `mvcc_experiment.sql` или два файла сессий;
- `secure_sql.md`;
- при необходимости `fixed_queries.sql`.

## Критерии зачета

- MVCC-наблюдения согласованы с порядком шагов транзакций.
- Исправленный SQL не собирает пользовательские значения строковой конкатенацией.
- Динамические идентификаторы проверяются через whitelist.
- Студент понимает, какие права нужны приложению и почему не стоит работать от имени владельца схемы.

## Контрольные вопросы: что вы должны уметь после занятия

1. Что видит транзакция до и после commit другой транзакции?
2. Почему MVCC не означает отсутствие конфликтов?
3. Как конфликтующее обновление проявляется в приложении?
4. Почему параметризация защищает значения, но не имена столбцов?
5. Как безопасно реализовать динамическую сортировку?
6. Какие права обычно нужны приложению для выполнения SQL-сценария?
