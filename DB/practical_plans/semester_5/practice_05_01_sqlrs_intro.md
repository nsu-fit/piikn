# Практическое занятие 5.1. Вход в Taidon/sqlrs и воспроизводимый SQL-сценарий

Место в курсе: первое практическое занятие 5-го семестра. Оно вводит рабочий формат практик: студент выполняет SQL-сценарий в учебной среде, фиксирует результат и сдает воспроизводимый артефакт.

Тип занятия: практика.

## Цели

- Проверить доступ к учебной среде Taidon/sqlrs.
- Показать, как оформляется сдача практических работ через репозиторий.
- Научить запускать готовый SQL-сценарий и фиксировать результат без скриншотов как основного артефакта.
- Зафиксировать базовые правила воспроизводимости: исходный SQL, ожидаемый результат, порядок запуска, commit hash.

## Входные условия

- У студента есть доступ к учебной среде и репозиторию для сдачи работ.
- Преподаватель выдает готовый SQL-сценарий и небольшой набор контрольных запросов.
- Знание SQL не предполагается глубже базового чтения команд.

## План занятия

1. Разбор структуры практического курса и формата сдачи.
2. Вход в Taidon/sqlrs, проверка доступности учебного контура.
3. Запуск готового SQL-сценария.
4. Повторный запуск или откат к исходному состоянию, если среда поддерживает изоляцию сессий.
5. Выполнение контрольных запросов и фиксация короткого результата в Markdown.
6. Создание структуры папки первой работы.
7. Commit артефактов и проверка, что преподаватель может воспроизвести работу по README.

## Задание

Студент должен выполнить выданный SQL-сценарий, сохранить его как исходный артефакт, выполнить 2-3 контрольных запроса и описать порядок воспроизведения.

Минимальный состав работы:

- `README.md` с порядком запуска;
- `script.sql` или ссылка на выданный сценарий;
- `checks.sql` с контрольными запросами;
- `answers.md` с кратким результатом выполнения и замеченными ошибками, если они возникли.

## Артефакт сдачи

Commit в репозитории студента с папкой `semester-5/practice-01-sqlrs-intro`.

## Критерии зачета

- Работа запускается в учебной среде без ручных неописанных действий.
- В репозитории есть SQL-файлы и короткое Markdown-описание результата.
- README достаточно конкретен, чтобы другой человек повторил запуск.
- Commit сделан до согласованного срока.
