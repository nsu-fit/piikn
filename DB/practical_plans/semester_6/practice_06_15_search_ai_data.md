# Практическое занятие 6.15. Поиск и AI-сценарии

Место в курсе: последняя обычная практика 6-го семестра. Она показывает, как данные для поиска и AI-сценариев требуют отдельного проектирования качества, обновления и прав доступа.

Тип занятия: практика.

## Цели

- Спроектировать полнотекстовый поиск по учебным текстовым данным, диагностическим событиям или документации.
- Обсудить vector search как опциональное расширение, а не обязательную технологию.
- Определить pipeline обновления поискового индекса.
- Зафиксировать ограничения качества данных, безопасности и объяснимости результата.

## Входные условия

- Есть текстовые данные: события диагностики, сообщения ошибок, run logs, документы или user-facing descriptions.
- Студент понимает разницу между основной БД и производным индексом.
- Преподаватель задает поисковые сценарии.

## План занятия

1. Разбор поисковых use cases.
2. Подготовка текстового поля или документа для индексации.
3. Полнотекстовый поиск в PostgreSQL или учебном поисковом контуре.
4. Оценка качества: найдено/не найдено, шум, ранжирование.
5. Обсуждение vector prototype: embeddings, обновление, приватность, стоимость.
6. Определение source of truth и derived index.
7. Описание pipeline обновления индекса.
8. Подготовка фрагмента для финальной защиты проекта.

## Задание

Студент должен подготовить search design slice.

Минимальный состав:

- описание поискового сценария;
- source data и derived index;
- SQL-прототип полнотекстового поиска или design sketch;
- 3-5 тестовых запросов пользователя;
- критерии качества;
- риски privacy/security/staleness.

## Артефакт сдачи

Папка `semester-6/practice-15-search-ai-data`:

- `search-design.md`;
- `search.sql` или `prototype.md`;
- `quality-notes.md`.

## Критерии зачета

- Поисковый сценарий конкретен и проверяем.
- Производный индекс не объявлен источником достоверных данных.
- Качество поиска оценивается на тестовых запросах.
- AI/vector-часть, если она есть, имеет явно описанные ограничения.

## Контрольные вопросы: что вы должны уметь после занятия

1. Чем source of truth отличается от поискового индекса?
2. Какие данные подходят для полнотекстового поиска?
3. Как проверить качество поискового результата?
4. Какие риски создает stale index?
5. Почему vector search не заменяет проектирование данных?
6. Какие ограничения безопасности важны для AI-сценариев?
