# Практическое занятие 6.9. Интеграция PostgreSQL с приложением

Место в курсе: занятие связывает БД с кодом приложения: транзакционные границы, слой доступа к данным, SQL-level tests и риск N+1.

Тип занятия: практика.

## Цели

- Подключить мини-приложение или test harness к учебному PostgreSQL-контуру.
- Найти слой приложения, который читает и изменяет данные в PostgreSQL.
- Описать транзакционные границы прикладного сценария.
- Определить, какие запросы должны быть параметризованы.
- Подготовить основу для миграционного checkpoint на неделе 10.

## Входные условия

- Студент понимает транзакции и безопасный SQL из 5-го семестра.
- Есть доступ к подготовленному учебному примеру на согласованном языке программирования.
- Преподаватель задает сценарий: создание заказа, регистрация события, изменение статуса, перевод денег или другой транзакционный use case.

## План занятия

1. Выбор прикладного сценария.
2. Настройка connection string и пула соединений.
3. Поиск кода, который читает или изменяет БД.
4. Выделение repository/query layer или аналогичной boundary.
5. Определение транзакционной границы.
6. Поиск потенциального N+1 или избыточных запросов.
7. Проверка prepared statements и параметризации SQL.
8. Формулировка теста слоя доступа к данным.

## Задание

Студент должен описать один прикладной сценарий работы с PostgreSQL.

Минимальный состав:

- краткое описание user/system action;
- какие таблицы участвуют;
- где начинается и заканчивается транзакция;
- какие запросы выполняются;
- какие ошибки должны обрабатываться;
- какой тест подтверждает корректность слоя доступа к данным.

## Артефакт сдачи

Папка `semester-6/practice-09-application-integration`:

- `data-access-scenario.md`;
- `test-sketch.md` или фрагмент теста;
- при необходимости ссылки на код.

## Критерии зачета

- Сценарий связан с реальной или учебной backend-операцией.
- Транзакционная граница описана явно.
- Студент видит, где SQL должен быть параметризован.
- Тест проверяет поведение, а не только наличие метода.

## Контрольные вопросы: что вы должны уметь после занятия

1. Где в приложении проходит граница работы с БД?
2. Как определить транзакционную границу для use case?
3. Почему N+1 является не только ORM-проблемой?
4. Какие ошибки БД должен обрабатывать application layer?
5. Что проверяет тест слоя доступа к данным?
6. Как код приложения влияет на будущую миграцию схемы?
