# Практическое занятие 6.1. Локальное или контейнерное развертывание PostgreSQL

Место в курсе: первое практическое занятие 6-го семестра. Оно создает обязательный учебный PostgreSQL-контур, в котором студенты будут исследовать внутреннее устройство, хранение, индексы, WAL, backup/restore, репликацию и эксплуатацию.

Тип занятия: практика.

## Цели

- Развернуть PostgreSQL локально или в контейнере.
- Создать учебную базу, пользователя приложения и пользователя для миграций.
- Подключиться через `psql` и выполнить smoke checks.
- Загрузить seed dataset для последующих практик.
- Зафиксировать воспроизводимый порядок запуска окружения.

## Входные условия

- У студента установлен Docker/Podman или локальный PostgreSQL, согласованный с преподавателем.
- Есть Git-репозиторий для сдачи практик.
- Преподаватель выдает seed dataset и минимальные требования к версии PostgreSQL.

## План занятия

1. Разбор требований к учебному PostgreSQL-контуру.
2. Создание `compose.yml` или локальной инструкции установки.
3. Запуск PostgreSQL и проверка доступности порта.
4. Создание базы данных и ролей: application, migration, read-only.
5. Подключение через `psql`.
6. Загрузка seed dataset.
7. Smoke checks: версия PostgreSQL, список баз, ключевые таблицы, количество строк.
8. Оформление README с порядком запуска и остановки.

## Задание

Студент должен подготовить воспроизводимый учебный PostgreSQL-контур.

Минимальный состав:

- способ запуска PostgreSQL;
- роли и база данных;
- seed dataset;
- smoke checks;
- инструкция остановки и очистки окружения;
- предупреждение о том, какие команды удаляют данные.

## Артефакт сдачи

Папка `semester-6/practice-01-postgres-local-setup`:

- `compose.yml` или `setup.md`;
- `init.sql`;
- `seed.sql` или ссылка на выданный dataset;
- `checks.sql`;
- `README.md`.

## Критерии зачета

- Окружение запускается по инструкции.
- База и роли создаются без ручных неописанных действий.
- Smoke checks подтверждают работоспособность.
- README явно описывает порядок запуска, остановки и очистки.
- Студент не использует суперпользователя для прикладных действий без необходимости.

3. Какие роли нужны для приложения, миграций и read-only доступа?
4. Как проверить, что seed dataset загружен?
5. Какие команды могут удалить данные окружения?
