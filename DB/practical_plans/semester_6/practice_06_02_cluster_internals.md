# Практическое занятие 6.2. Устройство PostgreSQL-кластера

Место в курсе: обычная практика с checkpoint-артефактом. Студент связывает лекцию о физическом хранении с наблюдаемыми объектами PostgreSQL: процессами, каталогом данных, relation files и системными каталогами.

Тип занятия: практика с checkpoint-артефактом. Вся пара не резервируется под контроль.

## Цели

- Найти каталог данных PostgreSQL и объяснить его структуру на безопасном уровне.
- Связать базу, схему, таблицу и relation file.
- Использовать системные каталоги для исследования объектов БД.
- Наблюдать backend-процессы и активные соединения.
- Подготовить карту кластера как вход в следующие практики.

## Входные условия

- Выполнена практика 6.1.
- Студент может подключиться к учебному PostgreSQL-контуру.
- Преподаватель задает список системных представлений и каталогов для чтения.

## План занятия

1. Разбор терминов: cluster, database, schema, relation, tablespace.
2. Поиск data directory в контейнере или локальной установке.
3. Наблюдение процессов PostgreSQL.
4. Запросы к `pg_database`, `pg_namespace`, `pg_class`, `pg_attribute`.
5. Связь OID/relation name/relation file через `pg_relation_filepath`.
6. Наблюдение активных соединений через `pg_stat_activity`.
7. Краткое обсуждение, какие файлы нельзя менять вручную.
8. Оформление `cluster-map.md`.

## Задание

Студент должен подготовить карту учебного PostgreSQL-кластера.

Обязательные элементы:

- как запущен кластер;
- где находится data directory;
- какие базы и схемы созданы;
- какие ключевые relation files соответствуют учебным таблицам;
- какие backend-процессы или соединения видны;
- какие действия с файлами кластера запрещены.

## Артефакт сдачи

Папка `semester-6/practice-02-cluster-internals`:

- `cluster-map.md`;
- `catalog_queries.sql`;
- короткий фрагмент вывода или таблица наблюдений.

## Критерии зачета

- Карта кластера соответствует реальному учебному окружению.
- SQL к системным каталогам можно выполнить повторно.
- Студент понимает разницу между логическим объектом и файлом хранения.
- В работе явно указано, что ручное изменение файлов кластера недопустимо.
