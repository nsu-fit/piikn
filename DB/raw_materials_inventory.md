# Инвентаризация сырьевых материалов для курса "Базы данных"

Документ фиксирует, какие материалы из `DB/raw/Рабочая группа РГБД 2025` можно использовать при подготовке курса, как именно их перерабатывать и куда они ложатся в текущую структуру практик, контрольных и преподавательских решений.

Эта папка рассматривается как **сырьевой архив**, а не как готовый студенческий репозиторий. Материалы нужно адаптировать к уже принятой логике курса:

1. 5-й семестр: применение СУБД и SQL в готовой учебной среде, без администрирования.
2. 6-й семестр: локальный или контейнерный PostgreSQL, эксплуатация, оптимизация, миграции и архитектурные решения.
3. Студенческий шаблон не должен включать преподавательские решения и сырые материалы без отбора.

## Общая оценка

| Группа материалов | Пригодность | Основное применение | Важные ограничения |
|---|---|---|---|
| SQL DQL / Oracle HR | Высокая | Банк SQL-задач для `SELECT`, join, aggregate, subquery, CTE | Диалект Oracle и схема HR; нужно переносить паттерны задач на PostgreSQL/flights или отдельный учебный датасет |
| Реляционная алгебра | Высокая | Практика 5.5, банк вариантов к контрольным, преподавательские решения | Нужно синхронизировать обозначения с текущими материалами и не перегружать студентов формализмом |
| ER/DDL/DML практические задания | Высокая | Пул предметных областей для ER, DDL, DML, нормализации и контрольных | Часть заданий предполагает проект с нуля; нужно отделить от нашего сценария работы с готовой демобазой |
| Контрольная работа N2 | Средняя-высокая | Пул предметных областей для контрольных и индивидуальных вариантов | Требуется нормализация формата вариантов и критериев сдачи |
| Оптимизация запросов | Средняя | Идеи для 6.3-6.5 и банка оптимизационных вариантов | Документ черновой, содержит TODO; нельзя переносить как готовое задание |
| Темы занятий / лабораторная сетка | Средняя | Исторический reference для структуры и прогрессии лабораторных | Порядок тем отличается от текущего курса; использовать только как источник идей |
| Пустые подпапки нормализации/преобразования моделей | Низкая сейчас | Резерв для будущего пополнения | В текущем состоянии содержат только служебные файлы |

## Правила переработки

1. Не переносить raw-документы в студенческий репозиторий целиком.
2. Сохранять ссылку на исходный файл в комментарии преподавательского банка или metadata.
3. Переписывать Oracle/MySQL-специфику в PostgreSQL.
4. Для SQL-задач на HR не копировать названия сущностей механически, если основной учебный датасет - flights/bookings.
5. Для проектных ER/DDL-заданий явно задавать:
   - артефакт сдачи;
   - ограничения;
   - проверочный SQL;
   - expected solution в `DB/exercise_solutions`.
6. Для контрольных делать варианты из общего пула, но не публиковать решения в студенческом шаблоне.

## Предлагаемый порядок следующего прохода

1. Извлечь из `SQL-HR-1` и `SQL-HR-2` 15-20 паттернов SQL-задач.
2. Переписать их под flights/bookings и разложить на:
   - базовый `SELECT`;
   - joins;
   - aggregates;
   - `EXISTS`/anti-join;
   - "найти максимум по группе".
3. Из `ER-диаграммы.docx` и `Контрольная работа №2/Варианты заданий.docx` собрать 8-12 предметных вариантов для ER/DDL/normalization.
4. Из `Задание на оптимизацию запросов.docx` сделать один полноценный вариант для `practice_06_05`.
5. Обновить `DB/exercises` и `DB/exercise_solutions` только после отбора и согласования конкретной группы задач.
