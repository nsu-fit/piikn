## `schema.sql`

```sql
CREATE SCHEMA IF NOT EXISTS course_lab;

DROP TABLE IF EXISTS course_lab.issue_comments;
DROP TABLE IF EXISTS course_lab.issue_tickets;
DROP TABLE IF EXISTS course_lab.issue_projects;

CREATE TABLE course_lab.issue_projects (
    project_id integer GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    code text NOT NULL UNIQUE,
    name text NOT NULL,
    CHECK (code ~ '^[A-Z0-9_]{2,12}$')
);

CREATE TABLE course_lab.issue_tickets (
    ticket_id integer GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    project_id integer NOT NULL REFERENCES course_lab.issue_projects(project_id),
    title text NOT NULL,
    status text NOT NULL DEFAULT 'open' CHECK (status IN ('open', 'in_progress', 'closed')),
    priority integer NOT NULL DEFAULT 3 CHECK (priority BETWEEN 1 AND 5),
    external_key text UNIQUE,
    created_at timestamp NOT NULL DEFAULT current_timestamp,
    closed_at timestamp,
    CHECK (
        (status = 'closed' AND closed_at IS NOT NULL)
        OR (status <> 'closed' AND closed_at IS NULL)
    )
);

CREATE TABLE course_lab.issue_comments (
    comment_id integer GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticket_id integer NOT NULL REFERENCES course_lab.issue_tickets(ticket_id) ON DELETE CASCADE,
    author text NOT NULL,
    body text NOT NULL,
    created_at timestamp NOT NULL DEFAULT current_timestamp
);
```

## `check_observations.md`

```md
# Check observations

- valid project accepted;
- invalid project code rejected;
- valid ticket accepted;
- invalid priority rejected;
- open ticket with `closed_at` rejected;
- closed ticket without `closed_at` rejected;
- comment for missing ticket rejected.
```

## `answers.md`

```md
# Answers

`CHECK` constraints encode project code format, ticket status, priority range and consistency between status and `closed_at`.
`external_key` is unique but nullable because external systems may not assign an id immediately.
Comments reference tickets through FK because a comment without a ticket is not meaningful.
Online schema changes, lock impact and rollback strategy are intentionally postponed to semester 6.
```
