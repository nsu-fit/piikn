# Решение 5.4. Контрольная 1: анализ схемы и предметной области

## Эталонная структура ответа

Правильная работа содержит:

1. Назначение каждой таблицы фрагмента.
2. PK/FK/candidate keys.
3. Кардинальности и optional/mandatory-участие.
4. Явные правила, выраженные схемой.
5. Гипотезы и спорные правила.
6. Интерпретацию диагностических выводов.
7. ER-фрагмент, согласованный с текстом.

## Пример для `ticket_flow`

- `bookings`: факт бронирования, агрегирует один или несколько билетов и хранит итоговую сумму.
- `tickets`: билет конкретного пассажира внутри бронирования.
- `segments`: сегмент перелета в составе билета, связывает билет и рейс, хранит класс обслуживания и цену.
- `flights`: конкретный рейс по маршруту в конкретное время.

Связи:

- `bookings.book_ref` -> `tickets.book_ref`: 1:N.
- `tickets.ticket_no` -> `segments.ticket_no`: 1:N.
- `flights.flight_id` -> `segments.flight_id`: 1:N.

Диагностика:

- `tickets_without_booking = 0`: нет билетов без бронирования.
- `segments_per_ticket_sample`: один билет может включать несколько сегментов.
- `segment_amount_by_fare_class`: диапазоны цен по классам выглядят правдоподобно, но не доказывают корректность всех тарифов.

## Пример для `boarding_flow`

- `duplicate_seat_on_flight = 0`: на текущих данных одно место одного рейса не выдано двум пассажирам.
- `boarding_pass_without_segment = 0`: нет посадочных талонов без соответствующего сегмента билета.
- `boarding_pass_seat_not_in_airplane = 0`: места из посадочных талонов есть в конфигурации самолета.

## Пример для `route_flow`

- `routes_without_airport = 0`: аэропорты маршрутов существуют в справочнике.
- `flights_per_route_sample`: один маршрут используется множеством рейсов.
- `suspicious_route_same_airport = 0`: нет маршрутов с одинаковым аэропортом отправления и прибытия.

## Критическая граница

"Нет нарушений" не равно "схема гарантирует невозможность нарушения". Это различие обязательно должно быть в ответе.
