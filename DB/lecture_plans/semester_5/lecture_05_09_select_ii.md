# Лекция 5.9. `SELECT` II: CTE, рекурсия и оконные функции

Место в курсе: лекция расширяет базовый SQL до запросов, которые используются в реальных отчетных и аналитических сценариях внутри OLTP-БД.

## Цели

- Разобрать обобщенные табличные выражения `WITH`.
- Показать рекурсивные CTE для иерархий и графоподобных структур.
- Научить использовать оконные функции без потери детализации строк.
- Подготовить студентов к запросам с рейтингами, running totals, интервалами и иерархиями.

## План лекции

1. Зачем нужны CTE: декомпозиция сложного запроса, читаемость, повторное использование промежуточного результата.
2. Обычные CTE и вложенные подзапросы: сходство, различия, влияние на читаемость.
3. Рекурсивные CTE: anchor query, recursive query, условие остановки.
4. Примеры рекурсии: организационная структура, дерево категорий, цепочка зависимостей, маршрут.
5. Оконные функции: отличие от агрегатов с `GROUP BY`.
6. `OVER`, `PARTITION BY`, `ORDER BY`, window frame.
7. Ранжирование: `row_number`, `rank`, `dense_rank`, `ntile`.
8. Навигационные функции: `lag`, `lead`, сравнение с предыдущей или следующей строкой.
9. Накопительные и скользящие агрегаты: running total, moving average.
10. Ограничения оконных функций: порядок вычисления, фильтрация по результату окна, читаемость.

## Ключевые понятия

CTE, рекурсивный запрос, anchor query, window function, partition, frame, ranking, `lag`, `lead`, running total.

## Связь с практикой

Практика посвящена CTE и оконным функциям: рейтинги, накопительные итоги, поиск интервалов, иерархии и сравнение с альтернативными формами запросов.

## Контрольные вопросы: что вы должны были узнать за эту лекцию

1. Для чего используется конструкция `WITH`?
2. Из каких частей состоит рекурсивный CTE?
3. Чем оконная функция отличается от агрегата с `GROUP BY`?
4. Что задают `PARTITION BY` и `ORDER BY` внутри `OVER`?
5. Когда применяются `lag` и `lead`?
6. Какие риски есть у рекурсивных запросов без корректного условия остановки?
