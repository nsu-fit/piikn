# Лекция 5.15. Безопасное использование SQL из приложения

Место в курсе: лекция связывает SQL с безопасной разработкой приложений.

## Цели

- Показать SQL injection как следствие смешения кода запроса и пользовательских данных.
- Научить отличать строковую сборку SQL от параметризованного запроса.
- Объяснить границы безопасного динамического SQL.
- Связать безопасность запросов с правами доступа и принципом минимальных привилегий.

## План лекции

1. Поверхность атаки приложения, работающего с БД: формы, API, импорт файлов, административные интерфейсы.
2. SQL injection: как пользовательский ввод превращается в часть SQL-кода.
3. Примеры последствий: обход фильтра, чтение чужих данных, изменение данных, разрушение схемы.
4. Параметризованные запросы: отделение текста запроса от значений.
5. Prepared statements и placeholders в разных технологических стеках.
6. Почему экранирование строк не является основной защитой.
7. Динамический SQL: когда он нужен и какие части нельзя параметризовать напрямую.
8. Белые списки для имен таблиц, столбцов, направлений сортировки и операторов.
9. Least privilege: приложение не должно подключаться к БД суперпользователем.
10. Проверка безопасности: code review, тестовые payload, анализ логов, запрет опасных паттернов.

## Ключевые понятия

SQL injection, параметризованный запрос, prepared statement, динамический SQL, escaping, whitelist, least privilege, права доступа приложения.

## Связь с практикой

Студенты исправляют небезопасные SQL-фрагменты: сравнивают строковую сборку запроса и параметризацию, затем объясняют, какие атаки становятся невозможными.

## Контрольные вопросы: что вы должны были узнать за эту лекцию

1. Почему строковая сборка SQL с пользовательским вводом опасна?
2. Как параметризованный запрос защищает от SQL injection?
3. Почему простое экранирование не должно быть основной стратегией защиты?
4. Какие элементы динамического SQL требуют белых списков?
5. Что означает принцип минимальных привилегий для приложения?
6. Какие признаки небезопасного SQL-паттерна нужно искать на code review?
