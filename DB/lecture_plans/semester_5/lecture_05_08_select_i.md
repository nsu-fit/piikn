# Лекция 5.8. `SELECT` I

Место в курсе: первая лекция по основному языку чтения данных из реляционной БД.

## Цели

- Сформировать уверенное понимание базовой структуры `SELECT`.
- Разобрать фильтрацию, сортировку, соединения, группировку, агрегаты и `HAVING`.
- Научить объяснять результат запроса, а не только получать его.
- Подготовить студентов к набору прикладных запросов к учебному датасету.

## План лекции

1. Логический порядок обработки `SELECT`: `FROM`, `WHERE`, `GROUP BY`, `HAVING`, `SELECT`, `ORDER BY`, `LIMIT`.
2. Фильтрация: операторы сравнения, `BETWEEN`, `IN`, `LIKE`, работа с `NULL`.
3. Сортировка и ограничение результата: `ORDER BY`, стабильность результата, `LIMIT/OFFSET`.
4. Соединения: `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`, `CROSS JOIN`.
5. Условия соединения: естественный ключ, внешний ключ, ошибка случайного декартова произведения.
6. Агрегаты: `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`; различие `COUNT(*)` и `COUNT(column)`.
7. `GROUP BY` и `HAVING`: группировка строк и фильтрация групп.
8. Подзапросы: скалярные, табличные, коррелированные; когда подзапрос улучшает читаемость.
9. Читаемость SQL: алиасы, форматирование, явные имена столбцов, проверка промежуточных результатов.
10. Типовые ошибки: фильтр по правой таблице после `LEFT JOIN`, агрегаты без понимания гранулярности, дубликаты после соединения.

## Ключевые понятия

`SELECT`, `WHERE`, `JOIN`, `GROUP BY`, агрегат, `HAVING`, подзапрос, гранулярность, `NULL`, логический порядок обработки.

## Связь с практикой

Студенты пишут прикладные запросы к учебному датасету: отчеты, выборки по нескольким таблицам, агрегаты и объяснение полученного результата.
