# Лекция 6.2. Физическое хранение

Место в курсе: лекция объясняет, как логические таблицы становятся страницами, файлами и записями на диске.

## Цели

- Показать физическую организацию PostgreSQL на уровне, достаточном для инженерной эксплуатации.
- Объяснить страницы, heap, tuple, TOAST и буферный кеш.
- Связать типы данных и форму строк с размером таблиц и индексами.
- Подготовить студентов к исследованию storage-следа сервиса выполнения SQL-заданий.

## План лекции

1. Логическая таблица и физическое хранение: почему это разные уровни.
2. Страница как базовая единица чтения и записи.
3. Heap-таблица: строки, версии строк, служебные поля, порядок хранения.
4. Переменная длина строк: влияние `text`, `jsonb`, массивов и nullable-столбцов.
5. TOAST: хранение больших значений вне основной строки.
6. Индекс как отдельная физическая структура, требующая места и обслуживания.
7. Buffer cache: почему повторный запрос может быть быстрее первого.
8. Стоимость ввода-вывода: последовательное чтение, случайное чтение, чтение из памяти.
9. Измерение размеров: `pg_relation_size`, `pg_total_relation_size`, размеры индексов.
10. Практические выводы для схемы: широкие строки, JSON payload, частые обновления, архивные таблицы.

## Ключевые понятия

Страница, heap, tuple, TOAST, buffer cache, relation size, index size, sequential I/O, random I/O, широкая строка.

## Связь с практикой

Студенты исследуют PostgreSQL-хранилище для сервиса выполнения SQL-заданий: размеры таблиц и индексов, `pg_relation_size`, влияние типов данных и JSON payload.
